## 搬瓦工 CN2 GIA 测试之前，先搞清楚线路分类

很多人说"我想买 CN2 GIA"，但搬瓦工的产品线里其实有好几个东西长得很像：

- **CN2 GT**：早期推出的 CN2 线路，现在已经比较拥堵，性价比不如 CN2 GIA，主要存在于入门套餐（CN2/KVM 系列）
- **CN2 GIA**：最高等级，DC9 机房（USCA_9）用的是纯 CN2 GIA，所有去往中国的流量都走电信 AS4809、移动 CMIN2、联通 AS10099 三网优化

说实话，DC6 和 DC9 在国内体验上差别不大，DC6 的带宽冗余更充足一些，DC9 则是老牌机房，路由策略稍有不同——DC9 出站流量会先经过中国电信网络再分发给其他运营商，而 DC6 直接在洛杉矶本地完成互联。晚高峰测试下来，两者丢包率都能做到接近 0%。

测 ping 主要看两个数字：延迟（ms）和丢包率（%）。国内到 DC6/DC9 的电信平均延迟大概在 160-200ms 这个区间，属于中美间能做到的正常水平。丢包率如果白天测是 0%，晚上高峰也能保持这个状态，说明这段时间线路状况不错，可以放心买。

### 2. 测下载速度

除了 ping，还可以直接用测速文件测一下下载速度。搬瓦工 DC6 和 DC9 都有官方演示站点，上面可以跑 SpeedTest 和 LookingGlass 路由测试，可以清楚看到去程和回程的路由跳数。

实测下来，DC6 在晚高峰从国内下载速度通常能稳在高位，很多地区甚至能跑满本地带宽上限。DC9 表现类似，移动和联通用户体验也不错，因为回程分别走了 CMIN2 和 AS10099 这两条精品线路。

### 3. 用 LookingGlass 确认路由

LookingGlass 工具可以做 traceroute，直观看到数据包从机房出发经过了哪些节点到达国内。如果你看到路由里有 `59.43.x.x` 或者 `202.97.x.x` 开头的节点，说明走的是 CN2 GIA/CN2 GT。纯 CN2 GIA 的路由通常非常干净，跳数少，没有绕路。
